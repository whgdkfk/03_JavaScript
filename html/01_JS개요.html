프로그래밍 언어

    프로그램(program)
    - 명령어의 나열(집합)

    프로그래밍(programming)
    - 명령어를 나열하는 것 == 명령어 작성
      (== 코딩)

    프로그래머(programer)  
    - 명령어를 작성하는 사람

    (참고) 개발자(devleoper)
    - 기획, 설계, 코딩, 테스트, 유지보수등
    개발과 관련된 모든일을 하는 사람

    프로그래밍 언어(programming language)
    - 컴퓨터한테 명령어를 전달할 수 있는 언어

    - 개발자가 알아볼 수 있는 형태로 작성
      (프로그래밍 언어 별로 형태/규칙이 정해져 있음)

      -> 컴퓨터가 해석할 수 있는 형태(2진법)으로 변환
      -> 명령어를 연산/수행

JavaScript

Script 언어

    기본 프로그램(브라우저)의 동작을
    사용자의 요구에 맞게 수행되도록 해주는 언어

    별도의 소스코드 컴파일(한 번에 이진법 변환)을 하지 않고
    (일괄 번역)

    인터프리터를 이용해 한 줄씩 번역, 수행하는 언어
    (실시간 통역)

    - 장점 : 컴파일에 소요되는 시간이 없음

    - 단점 : 프로그램 실행 시간이 상대적으로 느림

JavaScript란?

    웹 브라우저(크롬, 엣지, 웨일 등...)에서
    주로 사용하는 인터프리터 방식의 객체 지향 프로그래밍 언어.

    ECMA(European Computer Manufacturers Association, 유럽 컴퓨터 제조업 협회)
    Script 표준을 따르는 대표적인 웹 기술
    (JS 버전명 : ES5, ES6, ...)

JavaScript를 작성하는 방법

  1. 브라우저 개발자 도구 -> Console에 입력하기
    (JS 연습, 확인 등에 많이 사용하길 권장)
  2. HTML 문서 내부에 작성하는 방법(script 태그 이용) -> Internal 방식
  3. HTML 문서 외부에 존재하는 .js 파일을 연결해서 사용 -> External 방식 
            1) 원하는 위치에 .js 파일 생성
        2) 연결을 원하는 HTML 파일 body태그 제일 아랫쪽에
          script 태그를 작성 -> src 속성 추가
  4. 태그에 직접 JS 코드를 작성 -> inline 방식 (권장 X)
