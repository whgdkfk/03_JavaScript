전달 인자(Argument), 매개 변수(Parameter)

        * 전달 인자(Argument)
          - 함수 "호출" 시 () 안에 작성되는 값
          - 작성된 값은 함수의 매개 변수로 전달됨

        * 매개 변수(Parameter)
          - 함수 "선언" 시 () 안에 작성되는 변수
          - "호출" 시 전달되는 값(전달 인자)를 저장하기 위한 변수
          - 매개 변수는 함수의 지역 변수(함수에서만 사용 가능)

return(반환)

        - 함수 내에 작성하는 명령어
        - 현재 수행 중인 함수를 종료하고
          호출한 곳으로 돌아가라
        - return; -> 함수 종료, 호출한 곳으로 돌아감
        - return 값; -> 함수 종료, 호출한 곳으로 값을 가지고 돌아감

        (참고)
        함수 정의 시 return을 작성하지 않은 경우
        자동으로 함수 정의 끝 부분에 return 구문이 추가된다.

화살표 함수(Arrow Function)

        - 익명 함수를 =>(화살표) 기호를 이용해서 간단히 작성하는 방법

        익명 함수: function(){}

        1. 기본 형태: () => {}

        2. 매개 변수가 1개인 경우
           (e) => {}
           또는 
           e => {} // 매개 변수가 1개면 () 생략 가능

        3. 매개 변수가 여러 개인 경우
           (a, b, c) => {} // () 생략 불가

        4. 함수 정의 부분{} 내에 한 줄만 작성된 경우
           () => {console.log(123);}
           또는
           () => console.log(123); // {} 생략

           * {} 내 작성된 한 줄이 return 구문인 경우
           () => { return 1 + 2; }
           또는
           () => 1 + 2

        5. 함수 정의 부분에 return 한 줄만 있는데
           return되는 값이 객체 object 타입인 경우(객체)
           {}, return 생략이 불가능!
           () => { return {name:"홍길동"} }

function(){}, () => {} 차이점

        * this 참조 변수를 이벤트 핸들러 내부에서 사용 시 

        function(){}의 this === 이벤트가 발생한 요소
        () => {}의 this === window 객체

즉시 실행 함수

        - 익명 함수가 정의 되자마자 실행하게 만드는 함수

        [작성법]
        (function(){})();

        (()=>)();

        - 사용 이유
        1) 변수명 중복 문제 해결

        const str = "A"; // 전역 번수
        console.log(str); // A

        (function(){
            const str = "B"; // {} 내 지역 변수
            console.log(str); // B
        })(); 

        2) 상대적으로 성능(속도) 우위를 가짐

        1> 함수 정의(){}
        2> 함수()

        1> (function(){})()
